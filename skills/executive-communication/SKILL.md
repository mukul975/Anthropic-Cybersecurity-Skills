# Executive Communication Skill

Transform technical security findings into executive-appropriate language.

## Audience Profiles

### Board of Directors

- Focus: Fiduciary duty, regulatory compliance, reputation risk
- Format: Single-page summary, 3-5 key points
- Metrics: Dollars at risk, compliance percentages, peer comparison
- Tone: Professional, confident, clear recommendations

### Executive Leadership (CTO, COO)

- Focus: Operational impact, resource needs, strategic alignment
- Format: Concise with supporting detail available
- Metrics: SLAs, coverage percentages, trend lines
- Tone: Direct, business-focused, action-oriented

### Regulators

- Focus: Compliance posture, incident transparency, remediation
- Format: Formal, evidence-heavy, clear timelines
- Metrics: Control effectiveness, audit findings, remediation progress
- Tone: Professional, proactive, transparent

### Audit Committee

- Focus: Control effectiveness, risk management, compliance gaps
- Format: Structured by control domain
- Metrics: Finding counts, remediation status, trend analysis
- Tone: Balanced, evidence-based, forward-looking

## Translation Patterns

### Technical to Executive

| Technical Term         | Executive Translation                               |
| ---------------------- | --------------------------------------------------- |
| CVSS 9.8 vulnerability | Critical security gap requiring immediate attention |
| Lateral movement       | Attack spreading across systems                     |
| Zero-day               | Previously unknown attack method                    |
| EDR alert              | Security system detected suspicious activity        |
| Privilege escalation   | Attacker gaining administrative access              |
| MTTD                   | Time to detect threats                              |
| MTTR                   | Time to resolve security issues                     |
| Threat actor           | Attacker or criminal group                          |
| C2 infrastructure      | Attacker control systems                            |
| Exfiltration           | Data theft                                          |

### Metrics Translation

| Technical Metric       | Executive Metric                                 |
| ---------------------- | ------------------------------------------------ |
| MTTD: 4 hours          | We detect threats in under 4 hours               |
| 95% patch compliance   | 95% of systems are current on security updates   |
| 3 critical findings    | 3 issues requiring immediate executive attention |
| 99.9% uptime           | Less than 9 hours of downtime per year           |
| 15,000 blocked attacks | We stopped 15,000 attack attempts                |

## Output Templates

### Board Update (1 page)

1. **Security Posture Summary** (Green/Yellow/Red with trend arrow)
2. **Key Risks and Mitigations** (Top 3, business impact)
3. **Regulatory Compliance Status** (Compliance percentages)
4. **Notable Incidents** (if any, with resolution)
5. **Investment Needs** (upcoming budget considerations)

### Executive Briefing

1. **BLUF** (Bottom Line Up Front - one sentence)
2. **Situation** (What happened or what is the risk)
3. **Options with Trade-offs** (2-3 options, pros/cons)
4. **Recommendation** (Clear action)
5. **Resource Ask** (Budget, people, time needed)

### Regulatory Communication

1. **Subject Matter** (What this communication addresses)
2. **Current Status** (Compliance posture)
3. **Evidence of Compliance** (Specific controls in place)
4. **Remediation Plan** (if applicable, with milestones)
5. **Timeline and Commitments** (Clear dates)

### Budget Justification

1. **Investment Overview** (What and why)
2. **Risk Reduction** (Quantified where possible)
3. **Compliance Impact** (Regulatory requirements addressed)
4. **Competitive Position** (How peers are investing)
5. **ROI Analysis** (Cost avoidance, efficiency gains)
6. **Implementation Approach** (Phases, dependencies)

## Industry Context

Always consider:

- Regulatory and licensing implications
- Industry-specific compliance requirements
- Certification requirements
- Customer trust and brand reputation
- Responsible operations obligations
